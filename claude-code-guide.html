02 Introduction

What is Claude Code?

Your AI coding partner that lives in the terminal
  • It's a CLI tool — runs in your terminal on Mac, Windows, or Linux
  • Understands plain English — describe what you want, it writes the code
  • Takes real actions — reads, writes, and edits files and runs commands for you
  • Shows its work — every action is displayed before it runs
  • You stay in control — approve or deny any action at any time

04 Setup

Getting Started

You'll be ready in under 5 minutes
  1. 01
    Install Node.js
    Download from nodejs.org — this is the runtime Claude Code needs to run
  2. 02
    Install Claude Code
    In your terminal: npm install -g @anthropic-ai/claude-code
  3. 03
    Sign In
    Run claude — it opens your browser to log in with your Anthropic account
  4. 04
    Start a Session
    Navigate to any folder, type claude, press Enter. You're in.

07 Tool Calls

Reading Tool Calls

Claude shows its work — here's what you're seeing
// Claude shows these before running — you can approve or deny
Read → app.py // looking at your file
Edit → app.py line 42 // making a change
Bash → python app.py // running the code
Write → README.md // creating a new file
  • Read / Glob / Grep — exploring your codebase (safe, no changes)
  • Edit / Write — modifying files (review the diff shown)
  • Bash — running real commands (these execute for real — read carefully)

10 Intermediate

Working on Real Projects

When you're ready to go beyond single files
  • Navigate first — run cd your-project before starting Claude
  • Give context — "this is a React app for X, help me add Y"
  • Break it down — ask for one feature at a time, not everything at once
  • Review changes — Claude shows diffs; always read before approving file edits
  • Use git — commit working states so you can always roll back if something breaks
11 Teaching Claude

CLAUDE.md — Your Project Manual

A file that teaches Claude about your project — permanently
# CLAUDE.md — place in your project root folder
This is a Next.js blog. Pages are in /pages.
Always use TypeScript. Never use inline styles.
Run `npm test` before committing any changes.
Never log the API key stored in .env
  • Claude reads CLAUDE.md automatically at the start of every session
  • Write your rules, conventions, and context here — it never forgets them
  • A global one at ~/.claude/CLAUDE.md applies to all projects

13 Advanced

Agents & Subagents

Claude can spawn specialized helpers to work in parallel
  • An Agent is Claude spawning a second Claude to handle a subtask autonomously
  • Useful for big tasks — search the codebase AND run tests at the same time
  • Specialized types: Explore (search code), Plan (architecture), general (anything)
  • You don't trigger agents manually — Claude decides when they help

Think of agents as a team: Claude is the manager, subagents are specialists. You just give the goal.

15 Think Different

The Master Mindset

How experts think when working with Claude
  • Think in goals, not syntax — "I want users to log in" not "write a JWT function"
  • Iterate relentlessly — every response is a starting point, not a final answer
  • Read what Claude does — the best learning happens by watching it work
  • Teach Claude your standards — CLAUDE.md + feedback makes it smarter every session

The best developers don't write more code. They ask better questions and review outputs critically.
